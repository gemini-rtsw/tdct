Disclaimer: tdct on-line mode is an experimental feature. It It is meant to help the developer with testing of schematics.

Tdct can connect to EPICS IOCs and display online data for EPICS PVs. If configured, it can also modify PVs in an IOC.

Online mode can be entered from any online enabled tab by selecting the Online >> Go Online menu. The Online menu is only visible if the currently selected tab is online enabled.

A Tab is on-line enabled
  • when a schematics is displayed, which does not contain any macros
  • when a schematics is displayed, which was navigated to by descending the schematic hierarchy from an on-line enabled tab.
At the first attempt to enter online mode during a Tdct session, a dialog pops up for the user to confirm the EPICS ports. This is implemented as a safeguard to prevent writing inadvertently to a wrong IOC.

Display of EPICS PVs

In on-line mode, EPICS PVs are displayed the following way:
  • for each EPICS record, the VAL field is displayed and updated as the value changes in the IOC.

    It is colour-coded based on the records SEVR field. Default colour mapping is
    • NO_ALARM =>  blue
    • MINOR => yellow
    • MAJOR => red
    • INVALID => white
      Different colours may be defined through the [color] directive in the configuration file.
  • For  other EPICS record fields which are displayed on the schematics, tdct connects to the PVs only when  the online property sheet for the record is opened at the proper interest level.
    In this case the on-line value of the PV is displayed, otherwise "xxx" is displayed.
    If the online value is different from the value configured on the schematics (i.e. if the value was modified via Channel Access, dbpf, or by tdct), the value background is displayed in dark gray. This background color can be defined by a [color] directive in the configuration file.

  • If a property sheet is called up for an EPICS record, the field values are read from the EPICS IOC in the "value" column.
    The "edit" column next to the "value" column provides  text input fields or drop-down menus for modifying the field value for fields, which can be modified via Channel Access.

    Note: selecting a new value from a drop-down menu or terminating a text input field with RETURN will immediately write the new value to the IOC.


Writing to EPICS PVs

Writing to PVs must be enabled in the Tdct configuration file with the [options] write_to_ioc  directive.

On the property sheet for an EPICS record, the "edit" column next to the "value" column provides  text input fields or drop-down menus as appropriate for the field type. 

Note: selecting a new value from a drop-down menu or terminating a text input field with RETURN will immediately write the new value to the IOC.


EPICS Environment

The EPICS Channel Access ports can be configured in the Tdct configuration file with the [EPICS ports] directive.

If EPICS_CA_SERVER_PORT and EPICS_CA_REPEATER_PORT are defined as environment variables, they override the values from the configuration file.

If EPICS_CA_ADDR_LIST is defined as environment variable, it will be used in configuring the Channel Access context.

Before the first connect attempt of a Tdct Session, the ports need to be confirmed.
