Special Symbols

There are several "special symbols" which tdct recognizes by their names:
  • hardware input symbols which connect to an EPICS record input link field. 
    The name of a hardware input symbol must start with "hwin".
    The symbol must contain a  property (macro) where the property name starts with "val".
    The property value should be set via tdct to a hardware address or the name of a PV.
    Different hardware input symbols may be created with the tdct symbol editor, as long as they comply with the requirements above.

  • hardware output symbols which connect to an EPICS record output link field.
    The name of a hardware output symbol must start with "hwout". 
    The symbol must contain a  property (macro) where the property name starts with "val".
    The property value should be set via tdct to a hardware address or the name of a PV.
    Different hardware output symbols may be created with the tdct symbol editor, as long as they comply with the requirements above.

  • hierarchical connector symbols (input and output) which represent a symbol port on the corresponding schematics.
    The names of hierarchical connector symbols must start with "inhier" and "outhier".
    A tdct configuration file can only define one pair of hierarchical connectors.
    It is recommended to use only the names "inhier" and "outhier".
